Форк проекта https://github.com/huridocs/pdf-document-layout-analysis
Причины форка:
- нам нужна возможность использовать свои модели для парсинга таблиц и формул и гибко их переключать
- нужна поддержка разного бекенда для деплоя моделей - vllm, openrouter, ollama, llama cpp
- замены модели для ocr русского языка
- оптимизации
- кастомная логика
- в исходном проекте в Dockerfile зашивался код проекта, что мешало разрабатывать проект без постоянного ребилда. Теперь код проекта подгружается через volume, а в Dockerfile зашиваются внешние зависимости.
- исходный проект использовал Tesseract для ocr, что работает плохо для русского языка. Форк использует модель eslav из paddleocr как плагин для ocrmypdf.

Почему выбран именно pdf document layout analysis:
- ни один из сервисов не дает идеальное решение из коробки, форк одного из них - неизбежность
- чистая понятная архитектура проекта позволяет гибко развивать форк
- используется fastapi, что закладывает основу для проектирования сервиса как http сервиса
- в проекте используется одна из лучших opensource layout моделей
- на кейсе с borderless table даже без парсинга таблиц через отдельную модель получались корректные результаты (документ Концессионное соглашение - таблица с терминами - составные термины с переносами строк)
